* 시나리오 분석

>> 각 VOC 는 (분류정보(Category), Topic, Keywords, Keyword Set) 정보를 포함하고 있어야 한다.

1. 시나리오A

	- 검색 범위는 날짜 단위

	- 날짜별 상담내역(VOC) 분석결과 조회

	- 날짜 선택

	- 해당일에 수행된 상담내역(VOC)에 대한 카테고리 별 기본 통계 내역 보기(카테고리 별 상담내역 개수, 기타 정보)

	- 카테고리 선택 --> 카테고리에 포함된 하위 Topic 리스트: (Topic 리스트 화면에서 Topic 과 관련된 각종 통계 정보를 포함)

	- Topic 리스트 중 검색하고자 하는 Topic 선택

	- 선택한 Topic 에 포함되는 VOC 리스트 출력

	- VOC 선택 시 VOC 에 대한 Keyword Set 리스트를 출력하고 고객의 선택에 따라 VOC 원문 보기 지원


2. 시나리오B

	- 검색 범위는 날짜 단위

	- 검색을 통한 상담내역(VOC) 분석결과 조회

	- 검색 창에 검색어 입력: ex) 

	- 검색 결과 리스트/요약정보 화면: 카테고리별 요약, Topic 별 요약, Keyword Set 리스트 정보 포함

	- 각 결과를 선택 시 해당 정보 상세조회



3. 발표 시나리오

	* 개요

	- 전체 VOC 리스트 

	- 카테고리 분류: 주어진 개수에 따라 카테고리 분류(대분류)

	- 카테고리 별 Topic 리스트

	- Topic 에 포함된 VOC 리스트(?) or Keyword Set 리스트(?)


	* 시나리오 전개

	- 카테고리 분류 하기 --> 분류된 카테고리 정보

	- 카테고리 하위로 Topic 분석/추출 --> Topic 리스트

	- Topic 하위의 VOC 분석 --> Keywords / Keyword Set 리스트

        - 최종결과물: VOC 별로 카테고리 정보, Topic 정보, Keywords, Keyword Set 정보를 포함해야 한다. (각각의 정보는 중복 가능)
