        --> 상담STT파일 카테고리 분석
        
        --> 상담STT파일 키워드, 토픽 추출
        
        --> 상담STT파일 분석 시 불용어 처리문제 확인
        
    - 업무협의
    
        --> 참석: SKCNC(권오경GJ, 송희섭CJ 외 1명), Nextobe(이태희, 정윤수, 곽태민, 여진수)
        
        --> 고객상담내역STT파일 분석 시 키워드 및 토픽 추출에 대한 기술적인 이해상충 문제
        
        --> 고객상담내역STT파일 분석 시 잘 나타나지 않는 고객요구사항에 대한 통찰(Insight) 추출 문제
        
2. 이슈

    - 업무이슈
    
    - 기술이슈
    
        --> 고객상담내역STT 분석 시 마이너한 고객요구사항에 대한 통찰(Insight) 문제 해결 방안
        
            == SKCNC: 현재 시도중인 키워드, 토픽 추출 방식으로는 단위 Call 에 대한 숨겨진 요구사항을 통찰(Insight) 할 수 없음.
            
            == Nextobe: 고객이 요구하는 숨겨진 요구사항을 통찰(Insight) 해 내기 위해서는 단위 Call 정보만으로는 해결이 어려움
            
                ==> 진행 중인 단위 Call 상담 내역에 대해 카테고리 분류 및 키워드, 토픽 추출은 현재의 방법으로 계속 진행
                ==> 숨겨진 요구사항을 통찰(Insight) 하기 위해서는 주어진 Call 정보에서 마이너한 정보를 따로 관리하는 모델을 구성해서 별도로 처리할 필요가 있음.
                ==> 기존에 진행하던 Call 정보에 대한 분석은 현재 방식대로 고도화(案)
                ==> 고객의 숨겨진 요구사항을 통찰(Insight) 하는 문제는 전체 Call 정보를 대상으로 마이너한 정보를 토대로 추론을 할 수 있는 모델을 설계해 별도의 업무로 처리(案)
